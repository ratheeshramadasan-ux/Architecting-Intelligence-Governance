Stage 04 · Architect

AI Architecture & Implementation Control Plane

Translate governance requirements into enforceable identity, data, model, prompt, RAG, MCP, approval, evidence and monitoring controls.

Architecture principle

Governance states what must be controlled. Architecture defines where, how and by whom each control is enforced.

Control-plane layers

1. Identity & Access

User, service and agent identities; RBAC/ABAC; delegated authority; least privilege; step-up approval.

2. Data & Document Control

Classification, access inheritance, masking, tokenization, retention, lineage, document versions and deletion.

3. Prompt & Policy Gateway

Prompt enhancement, approved system instructions, policy enforcement, injection defence and secrets separation.

4. Model & Vendor Routing

Approved model registry, use-case routing, residency, contractual controls, fallback and cost management.

5. RAG & Knowledge

Approved sources, permissions, chunking, embeddings, citations, freshness, effective dates and obsolete-vector removal.

6. Agent & MCP Controls

Server registry, tool allow-lists, schema validation, identity propagation, action boundaries and sandboxing.

7. Validation & Human Approval

Automated tests, response validation, confidence thresholds, HITL, dual approval and exception routing.

8. Evidence & Audit

Model, prompt, SOP, sources, retrieval, tool calls, control decisions, approvals, final action and lineage.

9. Runtime Monitoring

Quality, safety, drift, access, cost, incidents, anomaly detection, kill switch, rollback and recovery.

Scenario implementation patterns

General and Conversational AI

Approved model, DLP, prompt policy, grounding where required, output validation and user accountability.

Document Understanding

Secure intake, OCR/ICR, classification, extraction, confidence, validation, HITL and source evidence.

Knowledge and RAG

Source ownership, versioning, access-aware retrieval, effective dates, re-embedding and citation verification.

Decision Support

Explainability, qualified human approval, outcome monitoring, appeal, recourse and audit reconstruction.

Agentic and MCP

Agent identity, tool authorization, transaction limits, pre-execution checks, approvals, rollback and kill switch.

Multi-agent Systems

Inter-agent identity, delegated authority, orchestration limits, loop detection and continuous assurance.

Production readiness gate

Design Evidence

Approved architecture, threat model, data flow, control mapping, vendor review and risk assessment.

Test Evidence

Functional, security, privacy, adversarial, performance, retrieval, regression and rollback tests.

Operational Evidence

Ownership, monitoring, support, incidents, DR, runbooks, access reviews and retirement plan.
